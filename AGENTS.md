# AGENTS.md — IMRecDeck

Instructions for any coding agent working in this repo. Written after a two-session
retrospective; most rules below exist because their absence cost a correction round.

## What this project is

Internal-medicine society guidelines → an Anki deck + a docs site, for **med students
and residents**. Live site: <https://cfu288.github.io/im-rec-deck/>. Repo:
`cfu288/im-rec-deck`.

## Pipeline map

```
manifest.yaml (human-edited — the ONLY human-authored input)
  → validate   scripts/validate_manifest.py         free
  → parse      scripts/parse_sources.py             free (epub/html local; PDF = LlamaParse, PAID)
  → build      scripts/build_references.py          free (skeleton; preserves enriched bodies)
  → enrich     scripts/enrich_references.py         PAID (Anthropic batch, Opus)
  → cards      scripts/generate_cards.py            PAID (Anthropic batch, Sonnet)
  → apkg       scripts/build_apkg.py                free (mega deck + per-guideline sub-decks)
  → docs       scripts/build_docs.py                free (Jekyll sources under docs/)
```

Orchestrated via the `Justfile` (`just --list`). **`just` may not be on PATH** — fall
back to `uv run scripts/<name>.py`, which is what every target runs anyway.
`just all-local` runs everything free; `enrich` and `cards` are deliberately excluded
from every `all-*` chain because they spend money.

## Generated vs. authored — edit generators, never output

- `manifest.yaml` — human-authored source of truth. Everything else derives from it.
- `references/guidelines/` — 100% generated (skeleton) + AI-enriched (paid). Tracked
  in git **deliberately**, because enrichment is expensive to regenerate.
- `docs/` — 100% generated by `build_docs.py` (except `_config.yml`, `_includes/`,
  `assets/`). Never hand-patch a generated page; fix the generator. A baseurl bug was
  once patched per-page instead of in the generator — don't repeat that.
- `build/` — generated artifacts (cards.jsonl, apkg files). Also tracked in git.
- `spec/knowledge-catalog/` — **vendored third-party git submodule** (Google's OKF
  spec, Apache-2.0). Its contents are not ours; don't audit, edit, or worry about
  files inside it.

## Idempotency contracts

Every script is safe to re-run; each skips work via a content hash:

| Script                  | Skip key                                                               | Notes                                                                                    |
| ----------------------- | ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| parse_sources           | provenance marker (source sha + best-format) in the `.md`              | epub > html > pdf; only re-parses on upgrade or source change                            |
| build_references        | body-content markers (`# Key Recommendations` etc.)                    | preserves enriched bodies + `_source_hash` on rebuild                                    |
| enrich_references       | `_source_hash` == sha of parsed source                                 | resumes interrupted batches from `build/.enrich-state.json`                              |
| generate_cards          | `body_hash` + `generator_version` per concept                          | purges + regens stale concepts; **retires** cards for concepts no longer `card_eligible` |
| build_apkg / build_docs | fully deterministic; stable GUIDs (sha1 of manifest keys) and deck IDs | apkg zip bytes churn on every run — content is stable, `git status` noise is expected    |

Sanity check after changing any generator: run it twice; the second run must produce
an empty `git diff`. Script output must be mdformat-canonical (blank line after
headings) or the pre-commit hook will churn every generated file.

## Supersession (adding a newer guideline version)

`current_version()` = latest dated year; yearless "living" topics use the **first
listed** version (manifest order is authoritative). Adding a newer version to a topic:

1. Drop source files in `tmp/`, then move to `sources/<system>/<topic>/<year>-<society>.{epub,pdf}`.
1. Add the version block at the top of the topic's `versions:` list in `manifest.yaml`.
1. Run: validate → parse → build → enrich (paid, ~$0.40/concept) → cards (paid,
   ~$0.02/concept) → apkg → docs.
1. Effects handled automatically: old version flips to `card_eligible: false`, its
   cards are retired from `cards.jsonl`, docs stop linking its sub-deck.
1. Manual step: delete the orphaned `build/decks/<system>/<topic>/<old-version>.apkg`.

Read the epub/PDF metadata (title in the OPF) **before** deciding which topic a new
file belongs to — filenames and DOIs mislead.

## Paid-batch policies (learned the expensive way)

- **Freeze inputs before submitting.** Never modify `sources/` or `references/` while
  a batch is in flight — a mid-flight source upgrade once staled 50/96 paid results.
- **Never cancel an in-flight batch to save money.** Batches process in seconds-to-
  minutes; the money is spent before a cancel lands. If budget is tight, ask before
  submitting, and cut scope by the `high_yield` manifest flag — never by accident of
  file state.
- **Estimates run low.** Historical misses: −36% (June 2026), −50% (July 2026).
  Include per-request system-prompt + tool-schema overhead; prefer the `count_tokens`
  API over chars-per-token heuristics. State the estimate and its basis before any
  spend, and calibrate from the actuals of prior runs when available.
- **Never write model/pricing constants from memory.** Fetch the official pricing
  page first; annotate constants with source URL + date verified.

## Site copy rules

Audience is med students / residents, not developers:

- No developer jargon: GUID, FSRS, canonical, hash, idempotent, repo mechanics.
  Say what the reader gets ("cards you've already reviewed keep your progress").
- No "attending will expect this" style commentary.
- Every Anki deck link gets the inline `anki.png` icon (see `ANKI_ICON` in
  `build_docs.py`; the banner button uses CSS `::before` in
  `docs/_includes/head_custom.html` because just-the-docs escapes HTML in `aux_links`).
- Deck download links point at raw GitHub:
  `https://github.com/cfu288/im-rec-deck/raw/main/build/...`.

## Verification habits

Verify at the layer the user experiences, not the layer you generated:

- Deck changes → the built `.apkg` must actually import into Anki (a TSV once failed
  653/653 notes on import despite being "well-formed").
- Docs changes → check rendered HTML (theme escaping differs from raw markdown);
  Jekyll build must pass before push — two deploys have failed on leftover files.
- After any rename/migration, grep for the old name (tags, paths, git-tracked files);
  two migrations left debris that the user had to find.

## Self-test and cost ledger

- `just check` (`scripts/check_repo.py`) — run after any generator or pipeline
  change, and before committing artifacts. Verifies: generator idempotency
  (double-run, byte-identical), enrichment preservation (enriched-body count
  never decreases), docs↔deck link integrity (including orphan sub-decks),
  manifest coverage (every topic resolves a current version; every enriched
  current version has cards), site-copy jargon lint, and an apkg import smoke
  test (sqlite note count + GUIDs vs `cards.jsonl`).
- `build/cost-history.jsonl` — both paid scripts append ACTUAL token usage and
  $ per ingested batch. Calibrate estimates from this file, not from priors.
- The `add-guideline` skill (`.claude/skills/add-guideline/`) is the runbook
  for ingesting a new guideline version end-to-end.

## Misc

- Python scripts are uv single-file scripts (`# /// script` headers); run with `uv run`.
- `.env` holds `ANTHROPIC_API_KEY` and `LLAMA_PARSE`.
- Pre-commit hook (`.githooks/pre-commit`) runs mdformat on staged `.md`, excluding
  `docs/` and `spec/knowledge-catalog/`.
- `build/.enrich-state.json` is the only intentionally untracked pipeline file.

## Incident log

Append three lines per notable incident (date, what happened, lesson). This
exists because a self-detected June accident was re-investigated as a mystery
in July — in-session knowledge evaporates; repo knowledge doesn't.

- **2026-06-22 — rmtree wipe.** `build_references.py` began with
  `shutil.rmtree(BUNDLE_ROOT)`; a routine study-guide regen wiped all 96
  enriched concept files (paid). Lesson: generators must preserve, never wipe;
  now enforced by the `preservation` check and the no-wipe note in
  `build_references.main()`.
- **2026-06-22 — mid-flight staling.** Sources were upgraded (epub over PDF)
  while a 96-request enrichment batch was in flight → 50 results stale on
  arrival. Lesson: freeze inputs before submitting a batch.
- **2026-06-23 — batch canceled under budget pressure.** An in-flight batch
  was canceled to save money; 92/96 had already processed and billed, and the
  resulting scope cut was arbitrary. Lesson: submitted batches are sunk cost;
  decide budget BEFORE submitting; cut scope by `high_yield`, never cancel.
- **2026-06-23 — TSV import failure.** First deck deliverable failed 653/653
  on Anki import (custom notetype). Lesson: verify the deliverable in its
  target app; now enforced by the `apkg` smoke test.
- **2026-06 + 2026-07 — cost estimates ran low.** −36% then −50%; both missed
  per-request system-prompt/tool-schema overhead and tokenizer density.
  Lesson: calibrate from `build/cost-history.jsonl` actuals.
- **2026-07-12 — orphaned sub-deck.** Superseding CHEST 2021 with the 2026 PE
  guideline left its sub-deck `.apkg` on disk and linked from docs. Lesson:
  supersession has a checklist (see `add-guideline` skill); now enforced by
  the `links` check.
